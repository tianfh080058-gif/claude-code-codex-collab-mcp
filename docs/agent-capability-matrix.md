# Agent Capability Matrix

Generated by Codex for Claude Code configuration governance.

Use this matrix to decide whether Claude Code CLI, Codex desktop, or both should handle a task.

## Default Routing

| Task Type | Primary Agent | Review Agent | Why |
| --- | --- | --- | --- |
| Product idea to PRD | Claude Code | Codex optional | Claude carries the user's product preferences and can structure requirements quickly. |
| Project code implementation | Claude Code | Codex for diff review | Claude runs inside the project workflow; Codex is strong at independent review. |
| Claude configuration governance | Codex | Claude Code for runtime sanity check | Codex is the auditor and backup/rollback planner for this `.claude` workspace. |
| Hooks, MCP, permissions, secrets | Codex | Claude Code only after explicit user approval | These are high-risk configuration surfaces. |
| Large refactors or multi-file docs | Codex | Claude Code for practical workflow impact | Codex handles broad file organization and documentation passes well. |
| Local CLI usability validation | Claude Code | Codex optional | Claude can validate whether runtime instructions and skills feel useful in CLI sessions. |
| Security review | Codex or `security-reviewer` | The other agent | Independent review helps catch permission, secret, and automation risk. |

## Codex Strengths

- Configuration audits and risk classification.
- Backups, rollback instructions, and change reports.
- Cross-file documentation and governance structure.
- Reviewing Claude runtime files without being the runtime agent.
- Detecting Codex/Claude instruction leakage.
- Conservative patch planning for hooks, MCP, permissions, and skills.

## Claude Code Strengths

- Working inside real project repos and existing developer workflows.
- Turning product intent into PRD, implementation paths, acceptance criteria, and validation steps.
- Using CLI skills, commands, subagents, and local project context.
- Generating task briefs, handoffs, review packs, and closeouts.
- Checking whether Codex-generated configuration actually helps the runtime experience.

## Collaboration Rules

- Use task archives under `reports/tasks/` for non-trivial work.
- Prefer one primary agent and one reviewer; avoid both agents editing the same files at the same time.
- Keep the user as dispatcher. A generated handoff or review pack is not execution approval.
- Sensitive work requires separate confirmation: secrets, `settings.local.json`, hooks, MCP, permissions, deletion, broad moves, or external services.
- Do not copy private values into task archives. Store only file, type, risk, and recommendation.

## Routing Heuristics

- If the task changes Claude Code configuration, start with Codex.
- If the task changes application code, start with Claude Code.
- If the task is unclear, start with `/task-brief`, then `/task-route`.
- If the task affects safety boundaries, ask Codex to review before implementation.
- If the task is complete, use `/task-closeout` before starting a new related task.
